# ピアノノート — プロダクト仕様(プロトタイプ準拠)

この文書は、HTML プロトタイプ `docs/prototype/etude-piano-app.html` から抽出した、Flutter 本実装の対象仕様。プロトタイプと本仕様が食い違う場合はプロトタイプを確認して本仕様を更新する。

## コンセプト

ピアノの練習と楽譜づくりを 1 アプリで行う「ピアノノート」。収録曲・自作曲の譜面を見ながら鍵盤で弾き、テンポ可変で再生し、楽譜エディタで音符を編集する。夜想曲をモチーフにした暗色＋真鍮色の UI。オフライン完結で、バックエンド・認証は持たない。

## 画面構成

4 画面。左サイドレール(ライブラリ / 練習 / 編集 / 演奏)で切り替える。レールはトグルで開閉できる。

1. **ライブラリ**(楽譜一覧)
2. **練習**(譜面＋鍵盤＋再生)
3. **楽譜エディタ**(音符の編集)
4. **自由演奏**(フリープレイ)

### ライブラリ

- **楽譜リスト**: 収録曲＋自作曲をすべて一覧表示する。各行に通し番号、曲名、作曲者、編集ボタン。曲をタップすると練習画面へ、編集ボタンで楽譜エディタへ。
  - 「今練習中」カード(習得度・最終練習日)は廃止した(ユーザー要望)。習得度(%)・最終練習日時・難易度(★)はモデルからも削除した(不要)。
- **楽譜を作成**ボタン: 空の自作曲(「無題の楽譜」)を新規作成し、楽譜エディタを開く。

### 練習画面

- **譜面**: ト音記号・拍子記号・小節線つきの五線譜。音符は拍位置 `t`・音価 `d` に応じて描画。再生中は playhead が右へ移動し、譜面が自動スクロールする。鳴っている音符はハイライトされる。停止中に音符をタップするとその位置を再生開始点に選べる(余白タップで先頭に戻す)。
- **鍵盤**: 横スクロール可能なピアノ鍵盤(白鍵・黒鍵)。タップで発音。再生中は対応する鍵盤が光る。
- **テンポ**: BPM スライダー(40–160)。再生中の変更は滑らかに追従する。
- **メトロノーム**: ON/OFF トグル。ON のとき拍頭(3 拍子なら 1 拍目)を強拍で刻む。
- **再生 / 停止**: 譜面を先頭から再生 / 停止する。
- **両手のお手本(任意)**: 両手フル譜面(`Piece.fullNotes`)を持つ収録曲だけ、AppBar のトグルで**大譜表(ト音＋ヘ音の2段)表示＋両手再生**に切り替えられる。表示専用(`GrandStaffView`)で、編集・開始位置選択はしない(鍵盤も隠す)。片手の単旋律(練習用)と切り替えて使う。現行の同梱曲は単旋律のみのため、このトグルは表示されない。
- **ヘッダー / サイドバー表示トグル**、この曲を**編集**へ遷移するボタン。

### 楽譜エディタ

- **曲名編集**: タイトル入力欄。空なら「無題の楽譜」。
- **音符ツール**: 音価選択(付点 2 分 / 2 分 / 4 分 / 8 分)、臨時記号(♯)トグル。
- **音符の追加**: 譜面上のタップ(拍位置と音高にスナップ)、または鍵盤タップ(挿入位置=キャレットの末尾に追加)。
- **音符の選択 / 編集**: 譜面上の音符タップで選択(強調表示)。選択中は音価変更・♯ 付与が選択音符に適用される。
- **削除**: 選択音符の削除、または末尾の削除。**全消去**。
- **試聴**: 編集中の譜面を再生 / 停止。
- **練習する**: 編集中の曲を練習画面で開く。
- 音符数の表示。挿入キャレット(点線)の表示。

### 自由演奏

- 音色セグメント(グランド / エレピ / ストリングス)。
- 横スクロールのフル鍵盤。タップで発音し、最後に弾いた音名を大きく表示する。

### 共通

- 音源の準備状況を示すステータス表示(「ピアノ音源を準備中…」→ 準備完了で自動的に消える)。
- 端末を横向きにすると鍵盤が広がる旨のヒント(プロトタイプの演出。本実装ではレスポンシブ方針として扱う)。

## 楽譜・音符モデル

- **音符** `Note`: 音高 `pitch`(`C4` / `F#5` などの科学的音名)、開始拍 `beat`(拍単位、0 始まり、double)、音価 `duration`(拍単位。8 分=0.5、4 分=1、2 分=2、付点 2 分=3)。
- **曲** `Piece`: id、曲名、作曲者、音符列 `notes`、拍子(`beatsPerMeasure`)、既定テンポ(`defaultBpm`)、自作フラグ。
- 調・拍子はプロトタイプでは**ハ長調・3/4 拍子固定**。本実装では曲のメタデータとして持ち、当面は同値を既定にする。
- 音高 ↔ 五線上の位置(diatonic step)変換、拍 ↔ X 座標変換などの**譜面ジオメトリは pure Dart のロジック**として持つ(プロトタイプの `dia()` / `noteY()` / `xAt()` 相当)。

## 再生ルール

- 各音符は `beat × (60 / bpm)` 秒の時刻に、`duration × (60 / bpm)` 秒だけ発音する。
- メトロノーム ON のとき、1 拍ごとにクリック音を鳴らす(拍頭は強拍)。
- 再生は曲の末尾(全音符の `beat + duration` の最大)＋わずかな余韻で自動停止する。
- 画面遷移・タブ切り替え時は再生を停止する。
- **再生スケジュールは実時間に依存させず、テストでは fake clock / `fake_async` で進める**(プロトタイプの `Tone.Transport` 直接依存は踏襲しない)。

## エディタの編集ルール

- 譜面タップ: X からスナップした拍(`curDur` 単位に丸め)、Y からスナップした音高に音符を追加。追加後はその音符を選択し、キャレットを次へ進める。
- 鍵盤タップ(エディタ時): キャレット位置にその音高の音符を追加し、キャレットを `curDur` 進める。
- 音符列は常に `beat` 昇順に整列する。
- ♯ トグルは黒鍵を持つ音名(C/D/F/G/A)にのみ有効。
- 全消去でキャレットと選択をリセットする。

## 状態・データ

### ライブラリ状態

- 収録曲(featured ＋ 既定の曲リスト)と、ユーザーが作成・編集した曲。
- 起動時の既定編集対象(featured)。

プロトタイプはメモリ保持のみ(リロードで消える)。**本実装ではユーザー楽譜(自作・編集)をローカル永続化する**(差し替え可能な `LibraryStore` 境界の対象。ADR 0001)。収録曲そのものは同梱データ(`ScoreRepository` 境界。ADR 0003)。

### 収録曲データ

- 収録曲: `{ id, title, composer, beatsPerMeasure, defaultBpm, notes[], fullNotes[] }`。既定で**パブリックドメイン(著作権満了)の童謡・唱歌・外国曲 11 曲**をシードする(featured = きらきら星)。
  - きらきら星 / ちょうちょう / かえるの合唱 / メリーさんの羊 / ロンドン橋 / ゆかいなまきば / 聖者の行進 / 茶色の小瓶 / アルプス一万尺 / ジングルベル / きよしこの夜
- 旋律(`notes`)は**手入力の片手・単旋律**。音域は練習鍵盤の C3–B5 に収め、音価は許容6種(16分/8分/4分/付点4分/2分/付点2分)のみを使う。
  - `fullNotes`(両手フル譜面)は現行データでは持たない(単旋律のため「両手のお手本」トグルは出ない)。スキーマとしては引き続き任意で持てる。
  - 曲追加時は著作権が満了した曲のみを対象とする(JASRAC 等の管理曲・近年の楽曲は不可)。
- データ不変条件(テストで保証する):
  - `notes` の `beat` は非負で昇順に並べられる(または並べ替え可能)。
  - `duration` は許容音価(0.25 / 0.5 / 1 / 1.5 / 2 / 3)のいずれか。
  - `pitch` は `^[A-G]#?\d$` にマッチする妥当な音名。
  - `id` は全曲で一意。`beatsPerMeasure` は 2–4、`defaultBpm` は 40–160、音域は鍵盤(C3–B5)内。

## 音源・再生エンジン

- 発音・再生は `AudioEngine` 境界(pure Dart)の裏に隠す。プロトタイプの Tone.js(Salamander サンプルピアノ＋フォールバックシンセ)に相当。
- 採用する音源プラグインの方針は ADR 0004。本番アダプタ(音を実際に鳴らす実装)は自動テスト対象外とし、テストでは呼び出しを記録する fake に差し替える。

## デザイン

- デザイントークン: 暗色 `--ink #141019` 系、生成り `--ivory #f2ead9`、真鍮 `--brass #d6a94c`、ばら `--rose #b67c86`(プロトタイプの `:root` 定義が正)。基調はダーク。
- フォント: 見出しは明朝(プロトタイプは Shippori Mincho / Fraunces)、本文はサンセリフ(Noto Sans JP)、数字・ラベルに装飾セリフ。スケルトンでは Shippori Mincho / Zen Kaku Gothic New / IBM Plex Mono を同梱(Fraunces 等のラテン装飾書体は UI 実装時に追加)。
- 五線譜・鍵盤は紙色(生成り)と黒鍵の対比で描画。
- アニメーション: 画面フェードイン、再生 playhead、鍵盤の glow。`prefers-reduced-motion`(Flutter では `MediaQuery.disableAnimations`)で抑制。
- アクセシビリティ: 鍵盤・音符・主要ボタンに意味ラベル(Flutter では Semantics)、トグルに pressed 状態、focus 可視化。

## スコープ外(プロトタイプ準拠)

- バックエンド通信・認証・課金。
- 複数 locale(日本語のみ)。
- オフライン同期(そもそも通信しない)。
- マイク入力・録音・採譜。
- MIDI 機器接続。
- 楽譜ファイルのインポート / エクスポート(MusicXML 等)。
